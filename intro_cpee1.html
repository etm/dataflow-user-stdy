On the left you can see the dataflow modeling extension for CPEE BPMN in comparison.

CPEE BPMN automatically layouts each modeling element (tasks, gateways, events), into its own line. IDs, labels and the dataflow are in the columns next to the diagram, associated with each line.

Regarding dataflow:

  • ▶ signifies that a task writes data.
  • ◀ signifies that a task reads data.
  • ● signifies that a task reads and writes data.
  • ▶, ◀ and ● symbols are on vertical bars ▮. Each vertical bar ▮ connects all reads and writes for a single data object for each task.
A CPEE BPMN is a dynamic diagram (i.e., interactively highlighted to help the user).
Hovering over a BPMN element, label, or id shows the name of all relevant data objects, if some exist . When you hover over an icon (▶, ◀ and ●), it shows the name of the selected data object . When you hover over a bar ▮, it shows not only the name of the selected data object , but it also highlights the bar ▮, so that you can see easily in which tasks the data object is involved. Try it!
